# 言外项目启动指南

## 环境准备

### 必需软件
1. **Node.js** (推荐 v16 或更高版本)
   - 下载地址: https://nodejs.org/
   - 安装后验证: `node --version`

2. **JDK 22** 
   - 下载地址: https://adoptium.net/
   - 安装后验证: `java -version`

3. **Maven** (可选，Spring Boot 项目可能包含 Maven Wrapper)
   - 下载地址: https://maven.apache.org/download.cgi
   - 安装后验证: `mvn --version`

4. **MySQL 8.0** (可选，用于真实后端)
   - 下载地址: https://dev.mysql.com/downloads/mysql/

### 可选方案
- **Docker + Docker Compose**: 一键启动所有服务

---

## 启动方式一：Mock 模式（推荐，最快）

使用项目自带的 Mock 服务器，无需数据库和真实后端：

### 1. 安装项目根目录依赖
```bash
cd AI_Chatbot_Project_Plan
npm install
```

### 2. 启动 Mock 后端服务器
```bash
node mock-server.js
```
Mock 服务器将在 http://localhost:8080 启动

### 3. 安装前端依赖并启动
新开一个终端：
```bash
cd yanwai-frontend
npm install
npm run dev
```
前端将在 http://localhost:3000 启动

---

## 启动方式二：完整模式（需要数据库）

### 1. 配置数据库
启动 MySQL 服务，执行数据库初始化脚本：
```bash
cd yanwai-backend/src/main/resources/sql
mysql -u root -p < init.sql
```

### 2. 配置后端环境变量
在 `yanwai-backend` 目录下创建 `.env` 文件（或设置环境变量）：
```
DB_URL=jdbc:mysql://localhost:3306/yanwai?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
DB_USERNAME=root
DB_PASSWORD=你的MySQL密码
YANWAI_AI_API_KEY=你的AI API密钥
YANWAI_AI_VISION_API_KEY=你的视觉AI API密钥
```

### 3. 启动后端
```bash
cd yanwai-backend
mvn clean package -DskipTests
java -jar target/yanwai-backend-1.0.0.jar
```

或直接运行：
```bash
cd yanwai-backend
mvn spring-boot:run
```

### 4. 启动前端
```bash
cd yanwai-frontend
npm install
npm run dev
```

### 5. 启动管理后台（可选）
```bash
cd yanwai-admin
npm install
npm run dev
```

管理后台将在 http://localhost:3001 启动

默认管理员账号：admin / 123456

---

## 启动方式三：Docker Compose（推荐）

确保已安装 Docker 和 Docker Compose：

### 1. 配置环境变量
在项目根目录创建 `.env` 文件：
```
AI_API_KEY=你的AI API密钥
AI_VISION_API_KEY=你的视觉AI API密钥
```

### 2. 一键启动所有服务
```bash
docker-compose up -d
```

服务地址：
- 前端: http://localhost:3000
- 后端: http://localhost:8080
- MySQL: localhost:3306

### 3. 启动管理后台（可选）
管理后台目前未包含在 Docker Compose 中，需要单独启动：
```bash
cd yanwai-admin
npm install
npm run dev
```

管理后台将在 http://localhost:3001 启动

默认管理员账号：admin / 123456

### 4. 停止服务
```bash
docker-compose down
```

---

## 验证启动成功

1. 访问 http://localhost:3000 看是否能打开登录页面
2. 使用访客模式或注册账号登录
3. 测试对话解码功能
4. （可选）访问 http://localhost:3001 打开管理后台，使用 admin/123456 登录

## 常见问题

### 前端无法连接后端
- 检查后端是否正常启动在 8080 端口
- 确认 `yanwai-frontend/.env` 文件中的 `VITE_API_BASE_URL` 配置正确

### 数据库连接失败
- 确认 MySQL 服务正在运行
- 检查用户名密码是否正确
- 确认数据库 `yanwai` 已创建

### 端口被占用
修改以下配置文件中的端口号：
- 后端: `yanwai-backend/src/main/resources/application.yml` 中的 `server.port`
- 前端: `yanwai-frontend/vite.config.js` 中的 `server.port`
- 管理后台: `yanwai-admin/vite.config.js` 中的 `server.port`
- Docker: `docker-compose.yml` 中的端口映射

### 管理后台无法登录
- 确认后端服务已启动
- 检查管理员表是否有数据（默认账号 admin/123456）
- 确认管理后台连接的后端地址正确
