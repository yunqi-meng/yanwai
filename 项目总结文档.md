# 言外 — 听懂TA的潜台词

## 项目总结文档

---

## 一、项目概述

**言外** 是一款基于 AI 深度语义分析的对话潜台词解读工具，同时内置了一套完整的游戏化人格卡牌收集系统和管理后台。用户输入对话文本（或上传聊天截图），系统即可对每句话进行情绪分析、潜台词解读，并给出沟通建议。每次分析完成后有机会随机获得人格卡牌，集齐碎片可合成完整卡牌或解锁成就，形成"分析-掉落-收集-成长"的产品闭环。

- **产品名称**：言外
- **产品定位**：AI 驱动的社交洞察 + 轻量化游戏体验
- **目标用户**：对人际沟通敏感、追求社交情绪价值的年轻用户群体（微信/QQ 聊天场景）
- **注册名称（软著方向）**：潜台词解码社交洞察游戏化软件 V1.0

## 四、数据库设计

共 **9 张业务表**，均使用 InnoDB + utf8mb4，MyBatis-Plus 统一管理逻辑删除（deleted 字段）。

| 表名                    | 说明         | 关键字段                                         |
|------------------------|-------------|------------------------------------------------|
| `user`                 | 用户表       | openid, member_level, daily_analysis_count, last_analysis_date, total_analysis, total_share, late_night_count, workplace_count, romance_count, login_days, legend_count, last_login_date |
| `analysis_record`      | 分析记录表   | user_id, original_text, analysis_result(JSON), relationship, emotion_curve(JSON), translations(JSON), advice(JSON) |
| `personality_card_def` | 卡牌定义表   | name, rarity(1-4), emoji, description, personality_type, trait |
| `user_card`            | 用户卡牌表   | user_id, card_id, quantity, is_new             |
| `user_card_fragment`   | 用户碎片表   | user_id, card_id, fragment_count               |
| `achievement_def`      | 成就定义表   | code, name, description, condition_field, condition_op, condition_value, reward_type, reward_value, icon |
| `user_achievement`     | 用户成就表   | user_id, achievement_id, unlocked_at           |
| `admin`                | 管理员表     | username, password(MD5), real_name, role, status |
| `system_config`        | 系统配置表   | config_key, config_value, description          |
| `operation_log`        | 操作日志表   | admin_id, operation, method, params, ip        |

**关键设计点**：
- 用户每天分析次数通过 `last_analysis_date` + `daily_analysis_count` 双重校验
- 会员到期自动降级（`member_expire_time` 过期 → `member_level` = 0）
- 成就系统使用 `condition_field` + `condition_op` + `condition_value` 动态匹配，支持 `>=`/`>`/`=`/`<`/`<=` 五种操作符
- 连续登录天数追踪（login_days）和传说卡牌收集统计（legend_count）
- 管理员密码使用 MD5 加密存储
- 系统配置支持动态修改，无需重启服务
- 操作日志记录管理员的所有操作，便于审计追踪

## 五、核心业务逻辑

### 5.1 对话分析流程（/api/analysis/decode）

```
用户发起分析
    │
    ├─ 校验用户是否存在
    ├─ 校验每日分析次数（会员跳过）
    │
    ▼
调用 AI API 分析（含 3 次重试 + 指数退避）
    │
    ├─ 真实模式：OkHttp → DeepSeek API → 解析 JSON 响应
    ├─ Mock 模式：MockAiService 本地生成模拟结果
    │
    ▼
保存 AnalysisRecord 到数据库
    │
    ├─ 更新用户统计：深夜次数 / 职场次数 / 情感次数 / 每日计数
    │
    ▼
卡牌掉落（30% 概率）
    │
    ├─ 普通用户概率：60%/25%/10%/5%（普通/稀有/史诗/传说）
    ├─ 会员概率翻倍：50%/25%/15%/10%
    ├─ 新卡牌：直接获得
    ├─ 重复卡牌：+1 数量，多出的转为碎片
    │
    ▼
成就检查（遍历全部 23 个成就）
    │
    ├─ 已解锁：跳过
    ├─ 未解锁：检查 condition 条件 → 满足则解锁
    │
    ▼
返回结果：分析内容 + 卡牌掉落 + 新成就
```

### 5.2 会员机制

- 免费用户：每日 3 次分析（`FREE_DAILY_ANALYSIS = 3`）
- 广告开通：每日观看广告可获得 24 小时会员资格（到期自动降级）
- 会员权益：每日无限分析 + 传说卡牌掉落概率翻倍（5%→10%）

### 5.3 行为统计与成就

系统自动追踪用户三类行为频率：
- **深夜解码**（22:00—06:00）：触发时段判定
- **职场分析**：对话包含工作/老板/同事等关键词
- **情感分析**：对话包含喜欢/恋爱/分手等关键词

23 个成就覆盖 8 个梯度，从"初窥门径"（分析1次）到"终极解密者"（分析1000次）。

## 九、AI 分析能力

### 9.1 分析维度

每次分析 AI 输出包含四个模块：

1. **关系识别**（relationship）：情侣/同事/朋友/家人等
2. **情绪曲线**（emotionCurve）：每句话的情绪评分(0-1) + 类型 + 描述
3. **逐句翻译**（translations）：原话 → 字面意思 → 潜台词 + 情绪评分
4. **沟通建议**（advice）：3-5 条针对性的改进建议

### 9.2 AI 接入方式

- 使用标准 OpenAI Chat Completions 格式
- 通过 OkHttp3 发起 HTTP POST，响应解析为 JSON
- 支持 3 次自动重试，指数退避（1000ms → 2000ms → 4000ms）
- 内置 Mock 模式：开发时无需真实 API Key，本地生成模拟分析结果

### 9.3 视觉 AI 支持

- 支持上传对话截图进行分析
- 通过视觉 AI 模型识别图片中的对话内容
- 默认使用豆包视觉模型 doubao-seed-1-6-vision-250815
- 可配置 vision-base-url、vision-api-key、vision-model 切换不同视觉模型

### 9.4 扩展性

通过修改 `application.yml` 中的 `base-url` 和 `model`，可切换至任意 OpenAI 兼容接口：
- DeepSeek（默认）
- 阿里通义千问
- 百度文心一言
- 智谱 ChatGLM

## 十、部署方案

### 10.1 Docker Compose（推荐）

```bash
# 一键启动 MySQL + Backend + Frontend
docker-compose up -d
```

三个容器编排：
- `yanwai-mysql`：MySQL 8.0 + 自动执行 init.sql
- `yanwai-backend`：SpringBoot 应用，连接 MySQL
- `yanwai-frontend`：Nginx 托管 Vue 产物，反向代理 /api 到后端

### 10.2 手动部署

**后端**：
```bash
cd yanwai-backend
mvn clean package -DskipTests
java -jar target/yanwai-backend-1.0.0.jar
```

**前端**：
```bash
cd yanwai-frontend
npm install
npm run build
# dist/ 部署到 Nginx
```

### 10.3 配置管理

所有敏感配置支持通过环境变量注入：
```bash
export AI_API_KEY=sk-xxx
export AI_BASE_URL=https://api.deepseek.com
export SPRING_DATASOURCE_PASSWORD=your_password
```

## 十二、项目亮点

1. **AI + 游戏化深度融合**：每次分析不仅是功能消费，更是一次"抽卡"体验，驱动用户持续使用
2. **动态成就匹配系统**：成就条件通过数据库字段化配置（condition_field + op + value），新增成就无需改代码
3. **广告变现闭环**：观看广告 → 24h 会员 → 无限分析 + 传说概率翻倍
4. **行为画像追踪**：深夜/职场/情感三类场景自动归类，连续登录统计，传说卡牌收集追踪
5. **视觉 AI 支持**：支持上传图片进行对话分析（视觉模型识别图片内容）
6. **多端支持**：Web 前端 + 管理后台 + 鸿蒙原生应用，覆盖多平台支持
7. **AI 接口可替换**：标准 OpenAI 格式，一行配置切换不同 AI 提供商
8. **完整管理后台**：数据看板、用户管理、内容管理、系统配置、操作日志，一站式运营管理
9. **一键部署**：Docker Compose 三容器编排，前后端数据库一体化
10. **Mock 开发模式**：无 API Key 也能完整体验全部功能
